Critical Security Assessment Case Study

Critical Security Assessment of an On-Premise EMR System

A real-world black-box external and physical security assessment on an enterprise infrastructure platform to identify vulnerabilities risking sensitive Patient Health Information (PHI).

Target Infrastructure:

On-Premise Enterprise Electronic Medical Record (EMR) Facility Server.

The Objective:

Conduct a black-box external and physical security assessment to identify vulnerabilities risking sensitive Patient Health Information (PHI).

Key Discoveries & Impact

  • Broken Authentication & Uncapped Brute-Force: The system authenticated users even when username capitalization was incorrect, exposing a flawed backend validation logic. Because password attempts were infinite (zero rate-limiting), the platform was entirely vulnerable to high-speed, automated brute-force attacks.
  • Cryptographic & Session Persistence Exploits: With brief physical machine access, testers easily exported client-side authentication certificates directly from the browser. These certificates lacked machine-binding, allowing attackers to clone them and gain persistent remote access from any external device globally.
  • Absent Audit Logging: The system failed to log remote authentication events on the central server. User sessions were only visible via local machine states, completely blinding the organization's security team to active network intrusions.

The Remediation (Our Fix)

  • Brute-Force & Access Hardening: Enforced strict, case-sensitive username validation and implemented rate-limiting with account lockouts after 5 failed login attempts.
  • Cryptographic Hardening: Reconfigured certificate deployment to utilize non-exportable private keys tied to specific workstation hardware security modules (TPM).
  • Centralized Logging: Integrated comprehensive logging to track all authentication traffic instantly across the entire on-premise network infrastructure.
